## 一、Blockly 简介

### 什么是 Blockly？

**Blockly** 是 Google 开发的一款可视化编程工具，它使用拖拽积木块的方式来编写程序，而不需要手写代码。

### 为什么选择 Blockly？

| 优势 | 说明 |
|------|------|
| 🎯 **零门槛** | 不需要学习编程语法，拖拽积木即可 |
| 🧩 **直观易懂** | 积木形状和颜色代表不同功能，一目了然 |
| ✅ **防错设计** | 不匹配的积木无法连接，避免语法错误 |
| 🔄 **即时转换** | 积木可实时转换为 Lua 代码 |
| 💾 **持久保存** | 项目可保存、导入、导出 |

### 在本项目中的应用

本项目的 Blockly 编辑器专为 QQ 机器人插件开发设计，提供了：
- 消息处理积木（接收/发送消息）
- 群管理积木（禁言、踢人、设置管理员等）
- 好友管理积木
- HTTP 请求积木
- 文件操作积木
- 数据存储积木
- 等等...

## 二、核心概念

### 1. 工作区（Workspace）

工作区是你拖拽和拼接积木的空白区域。

**操作方式：**
- **左键拖拽积木**：移动积木
- **右键长按拖拽**：移动工作区视角
- **滚轮**：缩放工作区
- **点击积木**：选中积木

### 2. 积木形状与类型

Blockly 有三种基本积木形状：

#### 🎩 事件积木（帽子形状）
```
┌─────────────┐
│  当收到消息时  │  ← 顶部圆弧，像帽子
└─────────────┘
       ↓
```
- **特点**：顶部圆弧，只能放在程序开头
- **作用**：程序的入口点，触发事件时执行
- **示例**：`当收到消息时`、`插件初始化时`

#### 🧩 语句积木（拼图形状）
```
       ↓
┌─────────────┐
│  发送群消息   │  ← 上下都有连接口
└─────────────┘
       ↓
```
- **特点**：上下凹凸，可垂直连接
- **作用**：执行具体动作
- **示例**：`发送消息`、`设置变量`、`日志输出`

#### 🔵 值积木（椭圆形状）
```
( 获取消息内容 )  ← 左右圆形接口
```
- **特点**：左右圆形，可嵌入输入框
- **作用**：返回一个值供其他积木使用
- **示例**：`获取消息的发送者ID`、`文本`、`数字`

### 3. 数据类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **字符串** | 文本数据 | `"你好"`、`"Hello"` |
| **数字** | 数值 | `123`、`3.14` |
| **布尔值** | 真/假 | `true`、`false` |
| **表/对象** | 键值对集合 | `{name: "小明", age: 18}` |
| **数组** | 有序列表 | `["a", "b", "c"]` |
| **空值** | 无值 | `nil` |

### 4. 变量

**变量**是存储数据的"盒子"，可以存放任何类型的数据。

```
设置 [用户名] 为 ["小明"]
        ↓
获取变量 [用户名]  →  返回 "小明"
```

**命名规则：**
- ✅ 可用：字母、数字、下划线（`user_name`、`count1`）
- ❌ 不可用：数字开头、特殊符号、空格（`1user`、`user-name`）

## 三、积木类型详解

### 事件积木

事件积木是程序的入口，当特定事件发生时自动执行。

| 积木名称 | 触发时机 | 用途 |
|---------|---------|------|
| `当收到消息时` | 收到任何消息 | 处理消息事件 |
| `当收到通知时` | 系统通知（进群/退群等） | 处理通知事件 |
| `当收到请求时` | 好友/加群申请 | 处理申请事件 |
| `插件初始化时` | 插件启动 | 初始化设置 |
| `插件卸载时` | 插件停止 | 清理资源 |

**使用示例：**
```
当收到消息时 存储到变量 [msg]
  ↓
  获取消息 [msg] 的纯文本内容
  ↓
  如果 内容 = "你好"
    发送私聊消息 ...
```

| true | true | true | true | false |

| false | false | false | false | true |

## 五、实战示例

### 示例 1：最简单的自动回复

**功能**：收到"你好"就回复"你好呀！"

```
当收到消息时 存储到变量 [msg]
  ↓
  如果 获取消息 [msg] 的纯文本内容 = "你好"
    ↓
    发送私聊消息 
        用户：获取消息 [msg] 的发送者ID 
        内容："你好呀！"
```

**生成的 Lua 代码：**
```lua
on_message(function(msg)
  if msg.get_plain_text(msg) == "你好" then
    message.send_private(msg.get_sender_id(msg), "你好呀！")
  end
end)
```

### 示例 2：群管机器人（关键词禁言）

**功能**：群里有人说脏话就禁言10分钟

```
当收到消息时 存储到变量 [msg]
  ↓
  如果 消息 [msg] 是群消息
    ↓
    如果 消息 [msg] 包含文字 "脏话"
      ↓
      禁言群 获取消息 [msg] 的群ID 
          用户 获取消息 [msg] 的发送者ID 
          时长 600 秒
      发送群消息 
          群：获取消息 [msg] 的群ID 
          内容："检测到违规内容，已禁言10分钟"
```

### 示例 3：积分系统

**功能**：记录用户发言次数

```
当收到消息时 存储到变量 [msg]
  ↓
  设置 [user_id] 为 获取消息 [msg] 的发送者ID
  设置 [key] 为 连接文本 "user_" 和 [user_id]
  设置 [count] 为 从存储读取 [key] 默认值：0
  设置 [new_count] 为 [count] + 1
  保存到存储 键：[key] 值：[new_count]
  ↓
  如果 [new_count] = 100
    发送私聊消息 
        用户：[user_id] 
        内容："恭喜！你的发言次数达到100次！"
```

### 示例 4：天气查询机器人

**功能**：发送"天气 北京"查询北京天气

```
当收到消息时 存储到变量 [msg]
  ↓
  设置 [text] 为 获取消息 [msg] 的纯文本内容
  ↓
  如果 消息 [msg] 包含文字 "天气 "
    ↓
    设置 [city] 为 截取文本 [text] 从 [4] 到 [文本 [text] 的长度]
    设置 [url] 为 连接文本 "https://api.weather.com/v1/current?city=" 和 [city]
    设置 [result] 为 HTTP GET请求 [url]
    ↓
    发送私聊消息 
        用户：获取消息 [msg] 的发送者ID 
        内容：连接文本 "当前温度：" 和 result.temperature
```

### 示例 5：群欢迎机器人

**功能**：新人进群时发送欢迎消息

```
当收到通知时 存储到变量 [notice]
  ↓
  如果 获取通知 [notice] 的 [通知类型] = "group_increase"
    ↓
    设置 [group_id] 为 获取通知 [notice] 的 [群ID]
    设置 [user_id] 为 获取通知 [notice] 的 [用户ID]
    发送群消息 
        群：[group_id] 
        内容：连接文本 "欢迎新人 @" 和 [user_id]
```

## 七、常见问题

### Q1: 为什么我的积木连不上？

**A:** 检查积木的形状是否匹配：
- 帽子积木只能放在最上面
- 拼图积木上下要对应
- 椭圆积木要放进对应的输入框

### Q2: 日志输出为什么是空白的？

**A:** 你可能直接输出了表对象。需要先用"将表转为JSON字符串"转换。

**错误：**
```
日志 (信息, msg)  ← 输出空白
```

**正确：**
```
日志 (信息, 将表 [msg] 转为JSON字符串)  ← 正常显示
```

### Q3: 如何判断是群消息还是私聊？

**A:** 有三种方法：

**方法1（推荐）：**
```
如果 消息 [msg] 是群消息
  ...
```

**方法2：**
```
如果 获取消息 [msg] 的 [消息类型] = "group"
  ...
```

### Q4: 如何获取发送者的昵称？

**A:**
```
获取消息 [msg] 的 [发送者昵称]
```

### Q5: 回复消息时引用消息ID有什么用？

**A:** 引用消息ID可以让回复显示为"回复某条消息"的样式。

- 填消息ID → 显示为回复样式
- 填 0 或不填 → 直接发送新消息

### Q6: 变量名有什么要求？

**A:**
- ✅ 可以用：字母、数字、下划线
- ✅ 示例：`user_name`、`count1`、`_temp`
- ❌ 不能用：数字开头、特殊符号、空格
- ❌ 错误：`1user`、`user-name`、`user name`

### Q7: 为什么我的数学计算结果不对？

**A:** 检查数据类型：
- `"123"`（字符串）不能参与数学运算
- `123`（数字）才能参与数学运算

**转换方法：**
```
设置 [num] 为 将 ["123"] 转为数字
→ num = 123
```

```
  从存储读取 ["mydata"] 默认值：["默认值"]
```

## 八、最佳实践

### ✅ 建议做法

1. **使用有意义的变量名**
   ```
   ✅ user_name, group_id, message_count
   ❌ x, y, z, a, b, c
   ```

2. **输出日志时使用JSON转换**
   ```
   ✅ 日志 (信息, 将表 [msg] 转为JSON字符串)
   ❌ 日志 (信息, msg)
   ```

3. **使用安全获取处理可能不存在的字段**
   ```
   ✅ 安全获取 [msg] 的 ["optional_field"] 默认值：[""]
   ❌ msg.optional_field  ← 可能报错
   ```

4. **添加注释说明程序逻辑**
   ```
   注释：这里是处理私聊消息的逻辑
   如果 消息 [msg] 是私聊消息
     ...
   ```

5. **使用常量代替魔法数字**
   ```
   ✅ 设置 [BAN_DURATION] 为 600  -- 禁言10分钟
      禁言群 [群号] 的用户 [QQ号] 时长 [BAN_DURATION] 秒
   
   ❌ 禁言群 [群号] 的用户 [QQ号] 时长 600 秒
   ```

6. **定期保存项目**
   - 使用 `Ctrl + S` 快捷键
   - 重要修改后立即保存

7. **分步骤测试**
   - 先实现核心功能
   - 逐步添加复杂逻辑
   - 每步都进行测试

### ❌ 避免做法

1. **直接输出表对象到日志**
   ```
   ❌ 日志 (信息, msg)  ← 输出空白
   ```

2. **不检查字段是否存在就直接使用**
   ```
   ❌ 直接使用 msg.group_id  ← 私聊消息没有这个字段
   ```

3. **混淆字符串和数字类型**
   ```
   ❌ "123" + 456  ← 错误！
   ✅ 将 ["123"] 转为数字 + 456  ← 正确：579
   ```

4. **使用过长的积木链**
   ```
   ❌ 一行里嵌套太多积木，难以阅读
   ✅ 使用变量拆分，每一步都清晰
   ```

5. **不处理错误情况**
   ```
   ❌ 直接发送HTTP请求，不管是否成功
   ✅ 使用带结果处理的HTTP积木，处理成功和失败情况
   ```
