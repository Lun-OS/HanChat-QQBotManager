# GMST 群消息同步插件

## 功能说明

GMST（Group Message Synchronization Tool）是一个群消息同步插件，用于将指定群的消息同步转发到其他群。

## 主要功能

1. **群消息监听**: 监听配置的源群列表中的消息
2. **消息转发**: 将收到的消息转发到其他配置的群（排除来源群）
3. **黑名单过滤**: 支持配置黑名单用户，黑名单用户的消息会被记录但不会转发
4. **消息日志**: 将所有消息信息记录到 `message.log` 文件中，包括：
   - 时间戳
   - 群号
   - 用户昵称和ID
   - 消息ID
   - 消息类型（text/image/at/face等）
   - 消息内容

## 配置说明

配置文件: `config.json`

```json
{
    "group_list": [
        1070163250,
        131933642,
        342584820
    ],
    "black_list_user_list": [
        3851157533,
        10000
    ]
}
```

### 配置项说明

- `group_list`: 源群列表，数组类型，包含需要监听和同步的群号
- `black_list_user_list`: 黑名单用户列表，数组类型，包含需要屏蔽的用户QQ号

## 使用方法

1. 确保插件目录存在 `main.lua` 和 `config.json` 文件
2. 在 `config.json` 中配置群列表和黑名单
3. 在插件管理界面加载插件
4. 插件会自动开始监听配置的群消息并转发

## 日志文件

插件会在插件目录下创建 `message.log` 文件，记录所有消息的详细信息。

日志格式：
```
[2024-01-01 12:00:00] Group: 1070163250 | User: 用户名 (123456789) | UserID: 123456789 | MessageID: 12345 | Type: text | Content: 消息内容
```

## 消息转发格式

转发消息格式：`[群 {群号}] {用户昵称}: {消息内容}`

例如：`[群 1070163250] 张三: 你好`

## 注意事项

1. 插件只会转发来自配置的源群列表中的消息
2. 黑名单用户的消息会被记录到日志，但不会转发
3. 消息不会转发回来源群（避免循环转发）
4. 如果只有一个配置的群，消息不会被转发（因为没有其他目标群）
5. 消息转发失败时会记录错误日志
