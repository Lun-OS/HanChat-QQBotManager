## 插件基础

### 什么是 Lua 插件

Lua 插件是基于 Lua 脚本语言编写的 QQBot 扩展模块，通过调用系统提供的 API 与 QQ 机器人进行交互，实现各种自动化功能。

### 插件特点

- **轻量级**: Lua 脚本语言简洁高效
- **热加载**: 支持动态加载和卸载
- **沙箱隔离**: 每个插件运行在独立环境中
- **账号隔离**: 插件数据按 QQ 账号分离存储
- **丰富 API**: 提供消息、群组、用户、文件、网络等全方位接口

## 插件目录结构

```
plugins/
├── {QQ号}/                          # 账号隔离目录
│   ├── {插件名称}/
│   │   ├── main.lua                 # 插件主文件（必需）
│   │   ├── config.json              # 插件配置文件（可选）
│   │   └── data.json                # 插件数据文件（自动创建）
│   └── ...
└── template/                        # 模板插件目录
    └── {插件名称}/
        ├── main.lua
        └── config.json
```

### 文件说明

| 文件 | 说明 | 必需 |
|------|------|------|
| `main.lua` | 插件主代码文件 | 是 |
| `config.json` | 插件配置，可在 Web 界面编辑 | 否 |
| `data.json` | 插件运行时数据存储 | 自动创建 |

## 插件生命周期

### 基本结构

```lua
-- 插件信息定义
plugin.name = "my_plugin"           -- 插件名称（唯一标识）
plugin.version = "1.0.0"            -- 插件版本
plugin.description = "插件描述"      -- 插件描述

-- 获取当前机器人 QQ 号
local selfId = plugin.self_id       -- 当前登录的机器人 QQ 号

-- 初始化函数（插件启动时调用）
function on_init()
    log.info("插件初始化")
    log.info("当前机器人QQ: " .. tostring(plugin.self_id))
end

-- 消息事件处理器
on_message(function(event)
    -- 处理消息事件
end)

-- 清理函数（插件停止时调用）
function on_destroy()
    log.info("插件停止")
end
```

### 插件信息字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `plugin.name` | string | 插件名称（唯一标识） |
| `plugin.version` | string | 插件版本号 |
| `plugin.description` | string | 插件描述 |
| `plugin.self_id` | number | 当前机器人 QQ 号（只读） |

### 生命周期流程

```
插件加载 → on_init() → 运行中 → on_destroy() → 插件卸载
              ↓           ↓
         注册事件处理器   接收事件回调
```

## 数据类型规范

### 统一 JSON 格式

项目采用 **JSON 格式** 作为唯一的数据交换标准，所有数据传递均使用 JSON 兼容的数据类型。

### 三种基础数据类型

| 类型名称 | 用途场景 | 示例 |
|----------|----------|------|
| **数字类型** | 数值计算、统计、ID、时间戳 | `123`、`45.67`、`-100` |
| **字符串类型** | 普通文本及 JSON 字符串 | `"hello"`、`"用户名"` |
| **布尔类型** | 开关状态、条件判断 | `true`、`false` |

### 复合数据类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **数组** | 有序元素集合 | `{"apple", "banana", "orange"}` |
| **对象/表** | 键值对集合 | `{name = "张三", level = 10}` |

### 数据类型转换

系统会自动处理以下类型转换：

| Go 类型 | Lua 类型 | 说明 |
|---------|----------|------|
| `int`, `int64`, `float64` | `number` | 数值类型 |
| `string` | `string` | 字符串 |
| `bool` | `boolean` | 布尔值 |
| `time.Time` | `number` | Unix 时间戳 |
| `[]interface{}` | `table` | 数组 |
| `map[string]interface{}` | `table` | 对象 |

### 最佳实践

```lua
-- ✅ 推荐：直接使用原生类型
storage.set("count", 100)
storage.set("enabled", true)
storage.set("user", {name = "张三", level = 10})

-- ✅ 推荐：读取时类型自动还原
local count = storage.get("count", 0)        -- 返回 number
local enabled = storage.get("enabled", false) -- 返回 boolean
local user = storage.get("user", {})          -- 返回 table

-- ✅ 推荐：事件数据直接使用
on_message(function(event)
    if event.message then
        for i, seg in ipairs(event.message) do
            if seg.type == "text" then
                log.info("文本: " .. tostring(seg.data.text))
            end
        end
    end
end)
```

```json
  "reply_text": "你好！我是机器人。",

  "admin_list": [123456789, 987654321]

```

```lua
-- 读取所有配置
local cfg = config.all()
local adminList = cfg.admin_list
```


**返回值**: 无

#### `message.send_group(group_id, message)`

发送群消息。

| 参数 | 类型 | 说明 |
|------|------|------|
| group_id | number | 群号 |
| message | string/table | 消息内容（支持 CQ 码或消息段数组） |

```lua
-- 发送消息段数组
local msgSegments = {
    {type = "text", data = {text = "你好"}},
```

### 存储 API

用于插件数据的持久化存储，数据以 JSON 格式保存在 `plugins/{QQ号}/{插件名}/data.json` 中。

**重要更新**：存储 API 现已支持任意 JSON 数据类型（字符串、数字、布尔值、数组、对象），不再局限于字符串。

#### `storage.set(key, value)`

存储数据，支持任意 JSON 可序列化的值。

| 参数 | 类型 | 说明 |
|------|------|------|
| key | string | 键名 |
| value | any | 值（支持字符串、数字、布尔值、表/对象、数组） |

```lua
-- 存储字符串
storage.set("counter", "100")

-- 存储数字
```

### HTTP 接口 API

插件可以注册自定义 HTTP 接口，供外部调用。

#### `http_interface.register(name, self_id, handler, methods)`

注册 HTTP 接口处理器（带账号隔离）。

| 参数 | 类型 | 说明 |
|------|------|------|
| name | string | 接口名称（URL 路径） |
| self_id | string/number | 当前机器人 QQ 号（用于账号隔离） |
| handler | function | 处理函数 |
| methods | table | 允许的 HTTP 方法（可选，默认 {"GET", "POST"}） |

**接口 URL 格式**: `/plugins/{self_id}/{name}`

```lua
-- 注册一个带账号隔离的 HTTP 接口
-- 访问路径: /plugins/123456789/my_api
local selfId = tostring(plugin.self_id)
http_interface.register("my_api", selfId, function(request)
    -- request 包含: method, path, query, headers, body
    log.info("收到请求: " .. request.method .. " " .. request.path)
    
    -- 返回响应
    return {
        status = 200,
        headers = {
            ["Content-Type"] = "application/json"
        },
        body = '{"success": true, "message": "Hello from plugin!"}'
    }
end, {"GET", "POST"})
```

#### `http_interface.unregister(name, self_id)`

注销 HTTP 接口。

```lua
local selfId = tostring(plugin.self_id)
http_interface.unregister("my_api", selfId)
```

#### 完整示例

```lua
plugin.name = "http_api_plugin"
plugin.version = "1.0.0"
plugin.description = "HTTP 接口示例插件"

local selfId = nil

function on_init()
    log.info("HTTP API 插件已启动")
    
    -- 获取机器人 QQ 号
    selfId = tostring(plugin.self_id)
    
    -- 注册状态查询接口
    http_interface.register("status", selfId, function(ctx)
        return {
            status = 200,
            headers = { ["Content-Type"] = "application/json" },
            body = '{"status": "running", "plugin": "http_api_plugin"}'
        }
    end, {"GET"})
    
    -- 注册数据处理接口
    http_interface.register("echo", selfId, function(ctx)
        -- 获取 POST 数据
        local body = ctx.body or ""
        return {
            status = 200,
            headers = { ["Content-Type"] = "application/json" },
            body = '{"echo": "' .. body .. '"}'
        }
    end, {"POST"})
    
    log.info("HTTP 接口已注册: /plugins/" .. selfId .. "/status")
    log.info("HTTP 接口已注册: /plugins/" .. selfId .. "/echo")
end

function on_destroy()
    -- 注销 HTTP 接口
    if selfId then
        http_interface.unregister("status", selfId)
        http_interface.unregister("echo", selfId)
    end
    log.info("HTTP API 插件已停止")
end
```

### 示例 2: 群管助手插件

```lua
plugin.name = "group_manager"
plugin.version = "1.0.0"
plugin.description = "群管助手 - 支持禁言、踢人、撤回等功能"

function on_init()
    log.info("群管助手已启动")
end

on_message(function(event)
    -- 只处理群消息
    if not msg.is_group_message(event) then
        return
    end
    
    local text = msg.get_plain_text(event)
    local groupId = msg.get_group_id(event)
    local senderId = msg.get_sender_id(event)
    local senderRole = event.sender and event.sender.role or "member"
    
    -- 解析命令
    local cmd, target, duration = string.match(text, "^#(禁言|解除禁言|踢人)%s*(%d*)%s*(%d*)")
    
    if cmd then
        -- 检查权限（只有群主和管理员可以使用）
        if senderRole ~= "owner" and senderRole ~= "admin" then
            message.send_group(groupId, "你没有权限使用此命令！")
            return
        end
        
        if cmd == "禁言" and target ~= "" then
            local seconds = tonumber(duration) or 600  -- 默认 10 分钟
            local success, err = group.set_ban(groupId, tonumber(target), seconds)
            if success then
                message.send_group(groupId, "已成功禁言 " .. target .. " " .. (seconds/60) .. " 分钟")
            else
                message.send_group(groupId, "禁言失败: " .. tostring(err))
            end
            
        elseif cmd == "解除禁言" and target ~= "" then
            local success, err = group.set_ban(groupId, tonumber(target), 0)
            if success then
                message.send_group(groupId, "已成功解除 " .. target .. " 的禁言")
            else
                message.send_group(groupId, "解除禁言失败: " .. tostring(err))
            end
            
        elseif cmd == "踢人" and target ~= "" then
            local success, err = group.kick(groupId, tonumber(target), false)
            if success then
                message.send_group(groupId, "已将 " .. target .. " 移出群聊")
            else
                message.send_group(groupId, "踢人失败: " .. tostring(err))
            end
        end
    end
    
    -- 撤回命令
    if text == "#撤回" then
        local replyId = msg.get_reply_id(event)
        if replyId then
            message.delete_msg(replyId)
        else
            message.send_group(groupId, "请回复要撤回的消息")
        end
    end
end)

function on_destroy()
    log.info("群管助手已停止")
end
```

### 示例 4: 使用 HTTP 接口的插件

```lua
plugin.name = "http_api_plugin"
plugin.version = "1.0.0"
plugin.description = "提供 HTTP 接口的示例插件"

local selfId = nil

function on_init()
    log.info("HTTP API 插件已启动")
    
    -- 获取当前机器人 QQ 号
    selfId = tostring(plugin.self_id)
    
    -- 注册 HTTP 接口（带账号隔离）
    -- 访问路径: /plugins/{selfId}/status
    http_interface.register("status", selfId, function(request)
        return {
            status = 200,
            headers = {
                ["Content-Type"] = "application/json"
            },
            body = '{"status": "running", "plugin": "http_api_plugin"}'
        }
    end, {"GET"})
    
    -- 注册数据处理接口
    -- 访问路径: /plugins/{selfId}/echo
    http_interface.register("echo", selfId, function(request)
        -- 获取 POST 数据
        local body = request.body or ""
        return {
            status = 200,
            headers = {
                ["Content-Type"] = "application/json"
            },
            body = '{"echo": "' .. body .. '"}'
        }
    end, {"POST"})
    
    log.info("HTTP 接口已注册: /plugins/" .. selfId .. "/status")
    log.info("HTTP 接口已注册: /plugins/" .. selfId .. "/echo")
end

on_message(function(event)
    -- 普通消息处理
end)

function on_destroy()
    -- 注销 HTTP 接口
    if selfId then
        http_interface.unregister("status", selfId)
        http_interface.unregister("echo", selfId)
    end
    log.info("HTTP API 插件已停止")
end
```

## 开发规范

### 命名规范

1. **插件名称**: 使用小写字母和下划线，如 `auto_reply`, `group_manager`
2. **变量命名**: 使用小驼峰命名法，如 `userId`, `groupList`
3. **常量命名**: 使用全大写和下划线，如 `MAX_RETRY`, `DB_FILE`

### 错误处理

始终检查 API 调用的返回值：

```lua
local success, result = message.send_group(groupId, "消息")
if not success then
    log.error("发送失败: " .. tostring(result))
end

-- 或者使用多返回值
local result, err = user.get_info(userId)
if err then
    log.error("获取信息失败: " .. tostring(err))
    return
end
```

### 日志规范

- 使用适当的日志级别
- 包含上下文信息
- 避免记录敏感信息

```lua
log.info("用户 " .. userId .. " 触发了关键词")
log.error("发送消息到群 " .. groupId .. " 失败: " .. errorMsg)
```

### 性能建议

1. **避免阻塞操作**: 长时间运行的任务应该分步执行
2. **缓存配置**: 频繁使用的配置项在 `on_init` 中读取并缓存
3. **限制存储大小**: 避免存储过大的数据
4. **合理使用事件**: 只注册需要的事件处理器

### 安全注意事项

1. **验证输入**: 对所有用户输入进行验证
2. **权限检查**: 执行管理操作前检查用户权限
3. **路径安全**: 文件操作被限制在插件目录内，不要尝试绕过
4. **敏感信息**: 不要将敏感信息硬编码在插件中

## 常见问题

### Q: 插件为什么不生效？

A: 检查以下几点：
1. 插件目录结构是否正确
2. `main.lua` 文件名是否正确
3. 插件是否在 Web 界面启用
4. 查看日志是否有错误信息

### Q: 如何获取消息中的图片？

A: 使用消息解析 API：
```lua
local hasImage = msg.has_image(event)
local imageUrls = msg.get_image_urls(event)
```

### Q: 如何回复被@的消息？

A: 检查是否被@并获取发送者信息：
```lua
if msg.is_at_bot(event, botId) then
    local senderId = msg.get_sender_id(event)
    message.send_group(groupId, "[CQ:at,qq=" .. senderId .. "] 你好！")
end
```

### Q: 配置文件修改后需要重启吗？

A: 不需要，配置修改后会自动生效。但建议在 `on_init` 中缓存配置以提高性能。
